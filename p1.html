Project 1: Full-Stack MERN Portfolio

A detailed guide to building a dynamic portfolio with React, Node.js, Express, and MongoDB.

Introduction & Architecture

Project Goal: To build a dynamic, database-driven portfolio. Instead of hard-coding your projects in HTML, you'll store them in a MongoDB database. A Node.js backend will serve this data via a REST API, and a React frontend will fetch and display it. The site will also feature a contact form that saves messages directly to your database.

Technologies Used: This is a full MERN stack project.

  • MongoDB: A NoSQL database to store project and contact information.
  • Express.js: A Node.js framework to build our backend REST API.
  • React: A JavaScript library to build our dynamic frontend user interface.
  • Node.js: The JavaScript runtime environment for our server.

(1) Frontend requests project data on page load.
(2) Server queries the database for all projects.
(3) Database returns the project documents.
(4) Server sends the project data back to the frontend as JSON.
(5) User submits the contact form; frontend sends data to the server.
(6) Server validates and saves the new message to the database.
(7) Database confirms the save operation.
(8) Server sends a success response back to the frontend.

Define Database Models

Mongoose uses schemas to define the structure of documents in a MongoDB collection.

models/projectModel.js

const mongoose = require('mongoose');
const projectSchema = new mongoose.Schema({
    title: { type: String, required: true },
    description: { type: String, required: true },
    technologies: [String],
    link: { type: String, required: true },
});
module.exports = mongoose.model('Project', projectSchema);

models/messageModel.js

const mongoose = require('mongoose');
const messageSchema = new mongoose.Schema({
    name: { type: String, required: true },
    email: { type: String, required: true },
    message: { type: String, required: true },
    createdAt: { type: Date, default: Date.now },
});
module.exports = mongoose.model('Message', messageSchema);
D

Build the API Routes

These routes define the endpoints our frontend will communicate with.

routes/api.js

const express = require('express');
const router = express.Router();
const Project = require('../models/projectModel');
const Message = require('../models/messageModel');

// GET all projects
router.get('/projects', async (req, res) => {
    try {
        const projects = await Project.find();
        res.json(projects);
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

// POST a new message
router.post('/messages', async (req, res) => {
    const message = new Message({
        name: req.body.name,
        email: req.body.email,
        message: req.body.message,
    });
    try {
        const newMessage = await message.save();
        res.status(201).json(newMessage);
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
});

module.exports = router;
E

Set Up the Express Server

This is the main file that ties everything together: connecting to the database, setting up middleware, and using our routes.

server.js

require('dotenv').config();
const express = require('express');
const mongoose = require('mongoose');
const cors = require('cors');

const app = express();
const PORT = process.env.PORT || 5000;

// Middleware
app.use(cors());
app.use(express.json());

// Database Connection
mongoose.connect(process.env.MONGO_URI, { useNewUrlParser: true, useUnifiedTopology: true })
    .then(() => console.log('MongoDB connected...'))
    .catch(err => console.log(err));

// API Routes
app.use('/api', require('./routes/api'));

app.listen(PORT, () => console.log(`Server running on port ${PORT}`));

.env (Environment Variables)

Create this file in the server root. Get your connection string from MongoDB Atlas.

MONGO_URI=your_mongodb_connection_string_here
PORT=5001

Step 3: Running & Deployment

You've built both the frontend and backend. Now let's run them together and get them on the internet.

A

Running Locally

You'll need two separate terminal windows.

Terminal 1: Start the Backend

cd server
node server.js

Terminal 2: Start the Frontend

cd client
npm run dev

Your React app should now be running (usually on http://localhost:5173) and successfully fetching data from your backend server (running on http://localhost:5001).

B

Deployment Strategy

We will deploy the frontend and backend separately. This is a standard practice for full-stack applications.

  • Backend (Render): Push your server folder to a GitHub repository. Sign up for Render, connect your GitHub, and create a new "Web Service". Point it to your backend repository. Render will detect it's a Node.js app. You must add your MONGO_URI as an environment variable in the Render dashboard.
  • Frontend (Vercel): Push your client folder to another GitHub repository. Sign up for Vercel, connect your GitHub, and import the client repository. Vercel will auto-detect it's a Vite/React app and deploy it.

Crucial Final Step: Once your backend is live on Render, it will have a public URL (e.g., https://my-portfolio-api.onrender.com). You must update the API_URL variable in your frontend App.jsx file to this new URL and redeploy the frontend. This tells your live frontend where to find your live backend.
